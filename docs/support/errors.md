# 错误码

NovaAPI 遵循标准 HTTP 状态码，错误详情在响应体中返回。

## 常见状态码

| 状态码 | 含义 | 排查建议 |
| --- | --- | --- |
| `400` | 请求参数错误 | 检查 JSON 格式与必填字段 |
| `401` | 鉴权失败 | 密钥缺失或无效，确认 `Authorization` 头 |
| `403` | 无权限 | 密钥无该模型/资源权限 |
| `404` | 资源不存在 | 检查端点路径与模型名 |
| `429` | 触发限流 | 降低并发或稍后重试 |
| `5xx` | 上游或网关错误 | 重试；持续出现请联系支持 |

## 错误响应示例

```json
{
  "error": {
    "message": "Invalid API key",
    "type": "invalid_request_error",
    "code": "invalid_api_key"
  }
}
```

排查不了时，请带上请求时间、模型名与错误信息[联系我们](/support/contact)。
